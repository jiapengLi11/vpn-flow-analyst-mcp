# VPN Flow Analyst 中文版

这个 Skill 用于指导 AI 完成网络流量研判任务，重点面向可疑加密隧道、VPN/代理类流量、flow 级风险解释和研判报告生成。

它不是检测模型本身，而是一份给 AI 的任务说明书：告诉 AI 面对流量分析问题时，应该先查什么、调用哪些工具、如何组织证据、怎样输出结论，以及哪些话不能乱说。

## 适用场景

当用户提出以下需求时，可以使用这个 Skill：

- 查询某条 flow 的风险信息
- 分析一条可疑网络会话为什么异常
- 解释命中特征和风险分数
- 检索 VPN / 代理 / 加密隧道相关知识
- 汇总一批可疑 flow
- 生成流量研判报告
- 判断是否可能存在误报

## 工作流程

### 1. 识别任务目标

先判断用户要分析的对象是什么：

- 单条 flow：用户提供了 `flow_id`
- flow 搜索：用户只提供了部分 flow 标识
- 批量汇总：用户希望查看高风险 flow 或某个风险阈值以上的会话
- 知识检索：用户询问协议特征、误报原因、指标解释或处置建议

### 2. 调用工具收集证据

根据任务类型调用 MCP 工具：

- `search_flows`：根据部分 flow_id 搜索候选 flow
- `analyze_flow`：分析单条 flow，获取风险分数、风险等级、命中特征和关键证据
- `search_vpn_knowledge`：检索协议特征、误报案例、指标说明和复核建议
- `generate_flow_report`：生成单条 flow 的结构化研判报告
- `summarize_flows`：按风险阈值汇总可疑 flow

### 3. 输出研判结论

输出内容应包含：

- 风险等级
- 风险分数
- 命中特征
- 关键证据
- 可能的误报原因
- 建议的下一步复核动作
- 当前结论的不确定性

## 输出格式建议

分析单条 flow 时，建议按下面结构输出：

```text
风险等级：
风险分数：
研判结论：
关键证据：
可能误报：
复核建议：
不确定性：
```

批量分析时，建议按下面结构输出：

```text
总体情况：
高风险 flow 数量：
主要命中特征：
典型样例：
优先复核建议：
```

## 安全边界

在安全流量研判中，AI 不能过度断言。

需要遵守以下规则：

- 不要声称已经解密 payload，除非工具结果明确提供了解密内容。
- 不要在证据不足时断言具体 VPN 类型。
- 如果结论来自统计特征，应使用“疑似”“可能”“表现出类似特征”等表述。
- 对正常下载、视频、CDN、大文件传输等可能误报场景，要主动提示复核。
- 不要输出攻击、绕过、规避检测或搭建违规代理的操作步骤。

推荐使用的稳妥表达：

- 疑似加密隧道流量
- 表现出 VPN-like 行为
- 命中特征显示其与普通 Web 流量存在差异
- 当前证据不足以确认具体协议类型
- 建议结合 SNI、域名类别、终端上下文和历史案例进一步复核

## 指标表述规范

如果需要说明模型效果或优化结果，必须写清楚：

- 对比基线是什么
- 使用的是哪个指标
- 是绝对提升还是相对提升
- 是否在同一验证集上比较

推荐表述：

```text
在同一验证集上，accuracy 从 72.35% 提升至 93.54%，绝对提升 21.19 个百分点；F1 从 0.7856 提升至 0.9545。
```

避免表述：

```text
准确率提升 23%。
```

因为这种说法没有说明是百分点提升，还是相对提升，容易被追问。

## 示例回答

当用户询问某条 flow 的风险时，可以这样回答：

```text
该 flow 当前被判定为中高风险可疑会话，主要依据是会话持续时间较长、上下行交互较均衡、未观察到明确 SNI，且风险分数较高。

从证据上看，它表现出一定的加密隧道特征，但当前证据不足以直接确认具体协议类型。考虑到大文件下载、视频流或 CDN 访问也可能产生类似包长和吞吐特征，建议进一步结合目的端口、域名类别、终端进程和历史访问行为进行复核。
```

## 面试解释口径

如果面试官问“Skill 是怎么做的”，可以这样回答：

```text
我把 Skill 设计成一个面向流量研判任务的 AI 工作说明书。它通过 frontmatter 定义名称和触发条件，在正文中约束分析流程：先判断用户是在查单条 flow、批量结果还是协议知识，再调用 MCP 工具获取 flow 特征、风险分数和知识库证据，最后输出风险等级、命中特征、误报可能和复核建议。

同时我在 Skill 中加入了安全边界，要求模型不能在证据不足时断言具体 VPN 类型，也不能声称解密 payload，而是使用“疑似加密隧道”“VPN-like 行为”等更稳妥的表达。
```
